Moving stuff around

Let's pass aditional uniform variable – u_time It is a cornerstone for animation (think of flowing lava textures in games for example) and as time is a shared value between all pixels, so that makes a perfect candidate for a uniform!

  • Try offsetting coordinate system center st by adding time
  • Try offsetting circle center by adding time
  • What if position of each pixel also affected how much time has passed for it specifically since shader program started being executed? Imagine each pixel position having it's personal clock.

TIP: Time makes a good candidate for sin() and cos() functions, to make looping animations!
TIP: To make modifications to u_time you have to assign it to another variable and then affecting that variable!
